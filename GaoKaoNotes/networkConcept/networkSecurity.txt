1. 網路安全設備
reference:
https://kknews.cc/zh-tw/news/gmzmomm.html
https://kknews.cc/zh-tw/tech/azkm2q6.amp
	A. 防火牆(firewall)  ----> 過濾封包
		a. 防火牆的功能:
			I. 被安置在內網和外網邊界
			II. 主要根據以下進行封包過濾:
				a. source IP
				b. destination IP
				c. protocol
				d. 時間time
				e. 動作
			III. 屬於主動防護(沒有攻擊活動, 他也有可能阻擋)
			IV. 主要運用在OSI 2~4層
		b. 防火牆分類
		reference:
		https://ithelp.ithome.com.tw/articles/10322987
			I. 軟體防火牆
				🤔. 由於設計硬體這件事是價格不斐的投資，拿著現有、通用硬體就能做事的軟體防火牆，就相對便宜得多了。像是我們一般人家中新台幣一兩千就能入手的路由器（Router），內建的就大多是軟體防火牆。
			II. 硬體防火牆
				🤔. 而「硬體防火牆」，就貴的多，一般也只有企業等級的防火牆會採用這樣的解決方案。相對於軟體來講，用硬體實現防火牆的優勢當然就在於性能高的多！像是分析封包、匹配規則等計算可以透過特殊設計的硬體來做，效率就會比通用的 CPU 做的運算高的多（如同 GPU 在圖形運算上遠快於 CPU 一樣）。

		c. 防火牆的多種不同政策(access policy)
			I. 基本存取: 允許內部網路和受信任的網路
			II. 全面管制: 限制所有流量
			III. 開放架構: 允許所有流量

	B. IDS(Intrusion-detection system, 入侵偵測系統),  ----> 監聽網路活動(封包可能已經進來內網了, 監視這些封包所造成的行為本身)
		a. 定義和功能:
			I. 是一種網路安全裝置或應用軟體，可以監控網路傳輸或者系統，檢查是否有可疑活動或者違反企業的政策
			II. 主要依靠監聽網路封包的形式，針對運行的狀況來進行監測

			III. IDS 檢測到潛在的入侵或安全事件後，會發出警報通知系統管理員，但本身不會主動採取防禦措施。屬於被動防禦


	C. IPS(Intrusion-prevention system)  ----> 監控封包內容(封包可能已經進入內網了), 若有可疑內容, 直接阻斷
		a. 定義和功能
			I. 其特點是可以分析到數據包的內容，解決傳統防火牆只能工作在4層以下的問題。

			II. 和IDS一樣，IPS也要像防病毒系統定義N種已知的攻擊模式，並主要通過模式匹配去阻斷非法訪問,致命缺點在於不能主動的學習攻擊方式，對於模式庫中不能識別出來的攻擊，默認策略是允許訪問的！

			III. IPS類設備，常被串接在主幹路上，對內外網異常流量進行監控處理，通常會串接在防火牆之後

			IV. IPS 具有阻止入侵的能力，可以自動執行防禦措施，例如封鎖攻擊流量、阻止攻擊者訪問受影響的系統等。屬於主動防禦
			
			V. 當然他的缺點就是需要的計算資源比IDS要更多

	D. WAF(Web application firewall)
		a. 定義和功能:
			I. 與傳統firewall 不同, 主要用在應用層, 針對HTTP應用做控制
			II. 應用程式層保護： WAF 位於應用程式層，能夠監控和檢測通過 Web 應用程式的所有請求和響應。它能夠阻止各種應用程式層攻擊，如SQL 注入、跨站腳本（XSS）、跨站請求偽造（CSRF）等。
			III. 訪問控制： WAF 可以根據預先定義的訪問規則來控制對 Web 應用程式的訪問。它可以基於用戶、IP 地址、地理位置等條件來控制訪問權限，從而防止未授權的訪問和惡意行為。
			IV. SSL/TLS 解密： WAF 可以對 SSL/TLS 加密的流量進行解密和檢測，從而檢測和防止加密通道中的安全威脅和攻擊。這使得 WAF 能夠全面保護 Web 應用程式，包括加密通道中的數據。
		

	F. 次世代防火牆(NGFW, Next-Generation Firewall)  ----> 現在市面上大份的防火牆都是這個了
	reference:
	https://www.cloudflare.com/zh-tw/learning/security/what-is-next-generation-firewall-ngfw/
		a. 定義:
			I. 通常使用了DPI(deep packet inspection, 深度封包檢測), 整合了IDS/IPS的功能, ssl/tsl加解密功能, 防禦能力比傳統防火牆高出不少, 同時整合了多項功能, 降低了維護的複雜性

2. 代理(proxy)
reference:
https://nordvpn.com/zh-tw/blog/vpn-proxy-bijiao/
https://ithelp.ithome.com.tw/articles/10202316
	A. 定義和功能:
		a. 定義:
			I. client發送封包至proxy, 再由proxy替client來發送封包至請求的目的網路server, server只會知道proxy的IP, 不會知道client的IP, 最後再由proxy響應回來
		b. 功能:
			I. 隱藏client的IP和資訊, 增加client的上網的隱蔽性
			

	B. 反向代理(reverse proxy)
		a. 定義:
			I. 與一般的proxy概念基本一致, 只是方向反過來, reverse proxy替server接收流量, 並替server發送封包給client, client只會知道reverse proxy的IP, 並以為reverse proxy的IP就是server的IP, client也不會知道reverse proxy如何把封包轉給真正的server(也無法知道)

		b. 功能:
			I. 附載均衡:
				🤔. 有些大行網站, 要處理的流量也多, 背後處理流量的server可能也多, 如何將流量盡可能平均的分配到各個機器, 此時reverse proxy就能發揮功用, 因為所有流量都會先經過它, 它正好可以控制流量的去處

			II. 避免直接遭受攻擊: 
				🤔. reverse proxy隱藏server真實的IP資訊, 降低hacking攻擊發生時對流量處理功能的損害. 這些攻擊會先針對reverse proxy, 避免了對真實用來處理流量的server直接傷害.

		c. reverse proxy server phishing(反向代理釣魚攻擊, 假冒網站)
			I. 做一個假冒網站server, 例如: phish-github.com
			II. 把phish-github.com的流量全部轉到github的reverse proxy, github會回傳畫面和功能
			III. 如此一來這個phish-github.com就成了一個"evil-reverse proxy", "假的reverse proxy"
			IV. 若有人使用了evil-reverse proxy登入github, 便可以偷偷把帳號密碼記下

3. vpn(virtual private network)
	A. 定義:
		a. 作用是在互聯網上的兩個端點(vpn endpoint)之間建立通道(tunnel)，以存取另一端點內的網路資源, 通道內流量全部加密, 接收端或傳送端自行加解密，可以將其視為虛擬專線 
			I. tunneling 技術

		b. 也就是說, 它是你可以自己架立(或說鋪設, 流量經過很多互聯網的節點)在全球最大的公開網路(public network) ,即互聯網(Internet) 的一種"私人網路", 故又稱為"虛擬私人網路"

		b. VPN 可以對資料進行加密，確保連線過程的安全性，如同一個架設在網際網路的虛擬加密資料通道(tunnl, 稱為隧道)。這表示即使網路服務供應商（ISP）監控使用者的網路活動，但 ISP 只知道使用者連上 VPN，無法看到使用者存取的網站內容。這種加密技術可以確保客戶端不被監控和追蹤，提供了線上隱私和安全的保障。
			I. tunnel(隧道)技術
			

	B. 功能
		a. 遠端安全性: 由於使用了加密 數據, 確保了數據傳輸過程的隱密性, 使用者可以透過在Internet建立tunnel傳輸私密數據(或者是企業組織的私密數據), 別人沒有機會得知數據內容.

		b. 隱私保護: 我們在一般的私人網路, 想要連線到互聯網Internet, 需透過gateway, 一般IP地址會經過轉換, , 外網的人不會知道你在私人網路內的IP地址, VPN是一樣的模式, 我們連線到VPN, 若是想要外網的內容, 數據會統一由VPN的gateway發出, 外網不會知道你的真實IP地址(但VPN內部是知道的, 所以只要該VPN有保留數據, 真的要查還是有機會查到的)
			I. 所謂翻牆其實就是利用了這個原理, 若有一個使用者其所在網路被限制訪問特定的外網, 此時若有VPN服務器公司提供VPN服務, 此VPN本身沒有被"牆"擋住, 那使用者便可連線到該VPN, 透過VPN連到欲訪問的外網.

			II. 這裡有一點需要注意, VPN的加密只對端到端, 它加密的對象是VPN_client <-> VPN_Server, 使用者若透過VPN連到外網, 這一段傳輸過程不是由VPN管理的, VPN並沒有對這一段加密:
				🤔. VPN_Client <---VPN加密---> VPN_Server <---無VPN加密---> 外網(Internet或是其他)

		
	C. VPN比proxy好在哪?
		a. client連線到proxy的過程並沒有加密(除非特別設置), 安全性較低, 且若訪問禁止的網站(例如被禁止的色情網站), 數據有機會在到達proxy前就被攔截阻擋, 但VPN不會, 因為全部加密過, 除了VPN_Server接收到數據並解密後能知道client欲訪問的網站, 其他人無法得知.

4. 防毒軟體
	A. 常見原理:
		a. 病毒簽名檢測(Virus Signature Detection): 
			I. 這是最常見的防毒技術之一。它基於事先收集到的已知病毒的特徵（即病毒簽名）來識別和檢測新的病毒。當防毒軟件發現文件或程式碼中包含與已知病毒簽名匹配的特徵時，它會標記該文件或程式碼為惡意

		b. 行為分析(Behavioral Analysis): 
			I. 文件變化監控： 行為分析可以監控文件系統中的異常變化。例如，如果某個程式突然開始向系統中的大量文件寫入數據，這可能是一個惡意活動的跡象，尤其是如果該程式沒有權限進行這樣的操作。
			II. 程式行為分析： 行為分析可以監控程式的執行行為。例如，如果一個程式開始嘗試修改系統設置、尋找敏感文件、或者嘗試與系統中的其他程式進行通信，而這些行為與該程式的正常功能不相符，這可能是一個惡意活動的跡象。
			III. 網路活動分析： 行為分析可以監控網路流量，檢測可能的惡意網絡活動。例如，如果一個程式開始向外部伺服器發送大量數據，或者嘗試與已知的惡意伺服器建立連接，這可能是一個惡意活動的跡象。
			IV. 系統設置變化： 行為分析可以監控系統設置的變化。例如，如果系統的安全設置突然被修改，或者新的用戶帳戶被創建，這可能是一個惡意活動的跡象

6. 黑客攻擊方法

		a. 惡意軟件攻擊（Malware Attacks）：這是一種常見的攻擊方式，包括病毒、木馬、間諜軟件（Spyware）和勒索軟件（Ransomware）等。黑客通過惡意軟件來入侵受害者的系統，竊取敏感信息、破壞系統運作或勒索金錢等。
		b. 釣魚攻擊（Phishing Attacks）：這種攻擊方式通常通過電子郵件、短信或社交媒體等途徑來進行，黑客偽裝成合法的機構或個人，試圖誘導用戶提供個人敏感信息，如用戶名、密碼、信用卡信息等。
		c. DDoS攻擊（Distributed Denial of Service Attacks）：這種攻擊方式旨在使目標系統或網站無法正常運行，黑客通過大量的流量或請求向目標系統發動攻擊，從而使其服務不可用。
			I. UDP Flood： 攻击者发送大量UDP数据包到目标系统，消耗其带宽和处理能力。
			II. SYN Flood： 攻击者发送大量TCP连接请求（SYN）到目标系统，但不完整地建立连接，导致目标系统资源耗尽。
			III. HTTP Flood： 攻击者发送大量HTTP请求到目标服务器，消耗其处理资源。
			IV. ICMP Flood： 攻击者发送大量ICMP Echo请求（ping），导致目标系统耗尽处理资源。
			V. Slowloris： 攻击者利用服务器对每个连接保持开放状态的特性，发送大量慢速的、未完成的HTTP请求，占用服务器资源。
			VI. DNS Amplification： 攻击者利用开放的DNS服务器向目标系统发送大量DNS查询请求，利用DNS响应的放大效应使得目标系统受到大量响应数据包的压力。
			V. NTP Amplification： 攻击者利用开放的NTP服务器向目标系统发送大量NTP查询请求，利用NTP响应的放大效应产生大量流量。
			VI. SSDP Amplification： 攻击者利用开放的SSDP服务向目标系统发送大量SSDP查询请求，利用SSDP响应的放大效应产生大量流量。

		d. 密碼破解攻擊（Brute Force Attacks）：黑客通過不斷地嘗試各種可能的密碼組合來破解受害者的密碼，以獲取未授權的訪問權限。
		e. 中間人攻擊（Man-in-the-Middle Attacks）：在這種攻擊中，黑客介入通信過程，窃聽、修改或偽裝通信內容，使得通信雙方誤以為他們正在直接通信。
		f. SQL注入攻擊（SQL Injection Attacks）：這種攻擊方式針對應用程序中的數據庫，黑客通過將惡意的SQL代碼插入到應用程序的輸入字段中，從而獲取敏感信息或對數據庫進行操作。
		g. 社會工程學攻擊（Social Engineering Attacks）：這是一種利用人們的社會和心理特徵來欺騙、誘導或操縱他們的攻擊方式，通常包括偽裝身份、欺騙、欺詐等手段。
		h. 帳號填充(Credential reuse): 這種攻擊是攻擊者透過遭駭網站所流出的帳號資料，嘗試使用這些帳號資料登入其他服務。當駭客利用系統漏洞取得某個網站的用戶帳號資訊，或者尋找已遭外洩的網站帳號資料庫後，就能利用這些帳號嘗試在其他網站上進行登入，以獲取其他網站上更多的機密資料。因此，不要在所有網站上都使用相同的密碼，以確保其他帳戶的安全。如果發現某個網站有帳號外流的新聞，請盡快更改該網站的密碼。
		i. 假冒無線熱點(Fake WAP): 這是一種非常簡單的駭客攻擊，很容易吸引受害者。飯店、咖啡廳或機場等公共場所的免費 Wi-Fi 熱點不一定安全，因為駭客可能會偽造這些場所的 Wi-Fi 熱點名稱，引誘人們連線。當有人連上這些假冒的 Wi-Fi 熱點後，駭客就能監控他們的網路活動，竊取機密資料，甚至強迫在設備上安裝惡意程式。
		j. 劫持網頁(web page hijacking): 用戶訪問的網站被重新定向到其他站點, 有或者竄改網頁內容, 內容可能包含虛假訊息, 誘騙用戶登入, 註冊, 導致用戶個人資訊洩漏, 有或者下載惡意程式,
			I. DNS劫持: 攻击者通过篡改DNS服务器的响应或在受害者设备上植入恶意软件，将合法网站的域名解析到恶意服务器上。这样用户在访问合法网站时会被重定向到攻击者控制的恶意网站。
			II. 中间人攻击（Man-in-the-Middle，MITM）： 攻击者位于用户和目标网站之间，窃取或篡改通信内容。通过欺骗用户，攻击者能够查看、修改或截取用户和网站之间的数据流量，从而实现劫持网页。
			III. 跨站脚本（Cross-Site Scripting，XSS）： 攻击者通过在受害者访问的网页中插入恶意的JavaScript代码，利用浏览器的安全漏洞执行恶意操作。这些恶意脚本可能用于修改网页内容、窃取用户信息或执行其他恶意行为。
			IV. 恶意重定向： 攻击者通过篡改网页中的链接或重定向规则，将用户重定向到攻击者控制的恶意网站。这种技术通常利用网站开发中的安全漏洞或未经授权的访问权限来实现。
			V. 会话劫持： 攻击者通过窃取用户的会话标识符或会话凭证，篡改用户的会话状态或冒充用户身份进行恶意操作。这可能导致攻击者接管用户的登录状态，执行未经授权的操作，或者以用户身份执行其他恶意行为。

7. IPSEC(Internet protocol security)   ---- 一套通訊協定(多個協定組成), 通常用於VPN, IPSEC與IP都屬網路層(使用時可選擇一個)
reference:
https://www.cloudflare.com/zh-tw/learning/network-layer/what-is-ipsec/
	A. 重要性:
		a. 安全通訊協定 (例如 IPsec) 是必要的，因為網路方法預設不會加密。例如通過郵政服務發送郵件時，通常不會在信封外面寫信件。 相反地，他們會將郵件包含在信封內，以便沒有人在寄件者和收件者之間處理郵件的人可以閱讀他們的郵件。 不過，像 TCP/IP 這樣的網路通訊協定套件只涉及連線和傳送，而且傳送的訊息不會隱藏。 中間的任何人都可以閱讀它們。 IPsec 和其他加密資料的通訊協定，在資料周遊網路時，基本上會在資料周圍放置一個封套，以確保資料的安全性。


	B. 定義:
		a. 是一種用於保護網際網路通信安全性的協議套件。它提供了對 IP 網絡上的數據進行加密和身份驗證的標準化機制，從而確保數據在傳輸過程中的機密性、完整性和可用性(CIA, confidentiality, integrity, authenticity)。

		b. 包含幾個主要協定(protocol):
			I. 驗證標頭(AH, Authentication Header):
				🤔. 身份驗證（Authentication）： AH 使用加密哈希函數對 IP 數據包的部分內容進行計算，生成一個摘要，稱為消息摘要碼（Message Digest）。此摘要碼用於驗證數據包的真實性，確保它沒有在傳輸過程中被修改或竄改。
				🤔. 完整性保護（Integrity Protection）： AH 通過在數據包的頭部和有效載荷（Payload）中添加驗證信息，來保護數據包的完整性。這樣，接收方就可以驗證數據包在傳輸過程中是否被修改。

			II. Encapsulating Security Payload (ESP)
				🤔. 加密（Encryption）： ESP 使用對稱密鑰加密算法對整個 IP 數據包進行加密，從而保護數據的機密性，防止攻擊者對數據的窺視和竊聽。
				🤔. 身份驗證（Authentication）： ESP 也可以提供對 IP 數據包的身份驗證，類似於 AH，但是它通常與加密一起使用，以確保數據包的完整性和真實性。 
				🤔. 完整性保護（Integrity Protection）： ESP 通常包含一個完整性校驗值（Integrity Check Value），用於驗證數據包在傳輸過程中是否被修改。這提供了對數據完整性的保護。

		c. 有兩種模式(mode)

			I. transport 
				🤔. 只會對封包的Data加密, 不會對header加密, IP_source和IP_destination不會被修改
			II. tunnel ---> VPN就是採用此法
				🤔. 會對整個IP 封包加密(包含Header), 並把gateway的IP作為新的header 裡面的IP
